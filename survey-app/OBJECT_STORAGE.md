## Этап 1. Аккаунт и каталог

1. Зайдите на [console.yandex.cloud](https://console.yandex.cloud).
2. Если облака нет — создайте **облако** и **каталог** (folder). Запомните **идентификатор каталога** (пригодится для CLI позже).
3. Убедитесь, что включена **оплата** (Object Storage обычно требует привязанный платёж).

**Промежуточный результат:** вы видите каталог в консоли, без ошибок доступа.

---

## Этап 2. Создать бакет

1. Меню **Object Storage** → **Создать бакет**.
2. **Имя** — глобально уникальное (латиница, цифры, дефисы), у вас сейчас: **`statisticsprimakov2`**.
3. **Класс хранилища** и **макс. размер** — по умолчанию для статики достаточно стандартных значений.
4. Регион **`ru-central1`**, если спрашивают.

**Промежуточный результат:** бакет появился в списке, внутри пока пусто.

---

## Этап 3. Хостинг статического сайта (обязательно для SPA)

Опросник на React — это **SPA**: при переходе на `/s/...` сервер должен отдать `index.html`, а не 404.

1. Откройте бакет → раздел **Настройки** / **Веб-сайт** (формулировка в UI может чуть отличаться).
2. Включите **хостинг статического сайта**.
3. Укажите:
   - **Главная страница:** `index.html`
   - **Страница ошибки:** тоже **`index.html`** (чтобы маршруты React работали при обновлении страницы).

Сохраните. Скопируйте из интерфейса **URL сайта** (website endpoint).

**Промежуточный результат:** по ссылке website может быть пусто или 403 — это нормально, пока нет файлов и прав.

## Этап 4. Публичное чтение объектов

Без доступа на чтение браузер не покажет сайт.

**Вариант A (через политику бакета — предпочтительно)**  
В настройках бакета найдите **политику доступа (bucket policy)** и добавьте правило, разрешающее всем читать объекты. Для бакета **`statisticsprimakov2`**:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::statisticsprimakov2/*"
    }
  ]
}
```

Если имя бакета другое — замените его в `Resource` в двух местах в строке ARN.

**Вариант B**  
При загрузке выставлять объектам ACL `public-read` (см. скрипт ниже, переменная `YC_OBJECT_ACL_PUBLIC_READ=1`).

**Промежуточный результат:** после заливки `index.html` сайт открывается в режиме инкогнито без авторизации.

## Этап 5. Сервисный аккаунт и ключи для загрузки

Загружать файлы с вашего Mac удобно через **AWS CLI** (S3-совместимый API).

1. **IAM** → **Сервисные аккаунты** → **Создать**.
2. Назначьте роль с правом заливать в бакет, например **`storage.editor`** (или более узкую по политике организации).
3. Для аккаунта создайте **статический ключ доступа** (Access key ID + Secret). **Секрет показывается один раз** — сохраните в менеджер паролей.

**Промежуточный результат:** у вас есть `AWS_ACCESS_KEY_ID` и `AWS_SECRET_ACCESS_KEY` (не коммитьте в git).

---

## Этап 6. Установить AWS CLI и проверить доступ

На Mac (Homebrew):

```bash
brew install awscli
aws --version
```

Проверка списка бакетов (ключи — из этапа 5):

```bash
export AWS_ACCESS_KEY_ID="..."
export AWS_SECRET_ACCESS_KEY="..."
export AWS_DEFAULT_REGION=ru-central1
aws s3 ls --endpoint-url https://storage.yandexcloud.net --region ru-central1
```

Если появится `Error: invalid option: --endpoint-url=...`: используйте **пробел** вместо `=` (как в строке выше), либо проверьте, что это именно AWS CLI:

```bash
which aws
aws --version
```

Должно быть что-то вроде `aws-cli/2.x`. Если `aws` оказался другой программой — переустановите: `brew reinstall awscli`.

**Промежуточный результат:** в выводе виден ваш бакет.

## Этап 7. Собрать фронт и залить

Из корня `survey-app`:

```bash
cd frontend
npm install
# Перед сборкой задайте URL API (иначе админка на сайте в бакете не достучится до бэкенда):
# • поле "apiBase" в survey-app/deploy.config.json, или
# • файл .env.production: VITE_API_BASE=https://... (шаблон: .env.production.example), или
# • VITE_API_BASE="https://..." npm run build
npm run build

chmod +x scripts/sync-object-storage.sh

export AWS_ACCESS_KEY_ID="..."

# Если без политики бакета — попробуйте:

./scripts/sync-object-storage.sh
```

**Промежуточный результат:** в консоли скрипт пишет `Синхронизация ... → s3://...`, затем `Готово`. Сайт по website-URL открывает опросник.

## API и CORS

Если фронт лежит на `*.website.yandexcloud.net`, а запросы идут на **другой домен** (API Gateway / функция), в настройках **CORS** бакета (или на стороне функции) разрешите origin сайта. В функции задайте переменную **`CORS_ORIGIN`** равной URL сайта (с `https://`).

## Частые проблемы

| Симптом | Что проверить |
|--------|----------------|
| 403 на сайте | Политика бакета или ACL объектов |
| Белый экран, в консоли 404 на `*.js` | Неверный `base` в Vite для вложенного пути; для корня бакета обычно достаточно `base: '/'` |
| React Router даёт 404 при обновлении | Страница ошибки хостинга должна быть `index.html` |
| `aws: command not found` | Установите `awscli` |
